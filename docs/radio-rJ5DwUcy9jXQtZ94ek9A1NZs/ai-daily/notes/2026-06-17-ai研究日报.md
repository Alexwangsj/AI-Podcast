1. 前沿模型公司正在从单纯发布模型，转向“模型 + 访问控制 + 交付生态 + 风险评估”的系统竞争。

3. 基础设施焦点从训练继续转向推理、数据中心融资、能源与合规约束。
4. 中国侧的重点是政策落地和产业融合：工信部“人工智能+信息通信”政策被地方周报和产业媒体持续转述，WAIC 2026 临近，国产模型、智能体和 AI 终端进入应用化阶段。
5. 研究前沿集中在部署前安全预测、科学仪器控制 agent、可复用技能树、物理 AI 世界模型、小模型蒸馏，以及评测与治理。

## 国际公司、模型与产品

### OpenAI：用部署模拟预测模型风险

- 来源：OpenAI 官方，Predicting model behavior before release by simulating deployment，2026-06-16  
  https://openai.com/index/deployment-simulation/
- 要点：
  - OpenAI 发布“Deployment Simulation”方法，用真实部署风格的历史对话前缀，在隐私保护后让候选模型重新生成响应，从而预测发布后的不良行为频率。
  - 文章称，该方法在多个 GPT-5 系列 Thinking 部署中改进了不良行为率估计，能帮助发现传统评测不容易覆盖的失配行为。
  - OpenAI 报告分析了约 130 万条去标识化对话，并把方法扩展到工具密集的 agentic coding 场景。
  - 该方法不替代红队、对抗评测和尾部风险分析，但增加了更接近真实部署的风险预测层。
- 解读：
  - 这是模型安全评估从“固定题库”走向“部署前仿真”的信号。
  - 当模型更像 agent，评测必须模拟工具、文件、网页、失败重试和长链路任务。

### LabOSBench：科学仪器 GUI 控制基准

- 来源：arXiv，LabOSBench: Benchmarking Computer Use Agents for Scientific Instrument Control  
  https://arxiv.org/html/2606.16802v1
- 要点：
  - 提出轻量可执行基准，评测 AI agent 对科学仪器 GUI 的控制能力。
  - 包含 8 个仪器模拟器和 96 个子任务，覆盖程序操作、视觉反馈和科学状态调整。
- 解读：
  - AI for science 不只是写论文摘要，也需要能操作实验软件、仪器界面和反馈循环。

### OpenClaw-Skill：自动构建可复用 agent 技能树

- 来源：arXiv，OpenClaw-Skill: Collective Skill Tree Search for Agentic Large Language Models，2026-06-15  
  https://arxiv.org/abs/2606.16774
- 要点：
  - 提出 Collective Skill Tree Search，通过多模型生成和评估技能节点，构建结构化、可迁移的技能树。
  - 目标是提高 LLM agent 在工具使用、多步推理和动态环境中的表现。
- 解读：
  - Agent 能力不只来自模型权重，还来自可复用技能、经验组织和训练数据闭环。

### Kairos：面向物理 AI 的世界模型栈

- 来源：arXiv，Kairos: A Native World Model Stack for Physical AI，2026-06-15  
  https://arxiv.org/abs/2606.16533
- 要点：

### Quality-Utility Paradox：小模型蒸馏中的高分数据陷阱

- 来源：arXiv，The Quality-Utility Paradox，2026-06-15  
  https://arxiv.org/abs/2606.16152
- 要点：
  - 论文认为，由强模型或 Oracle 精修后获得更高 reward 分数的数据，不一定更适合训练小模型数学推理。
  - 原因是高质量数据可能偏离小模型自身推理分布，增加适配成本。
  - 作者提出 Style-Aligned Refinement，在保留逻辑修复的同时保留小模型原生推理风格。
- 解读：
  - 小模型训练不能只追求“教师答案质量”，还要考虑学习者分布兼容性。
