## 重点事实与来源

### 1. OpenAI：ChatGPT Scheduled tasks 更新，Pulse 逐步退出

- OpenAI Help Center 的 ChatGPT Release Notes 显示，2026-06-17 更新了 Scheduled tasks。
- 新增侧边栏里的 Scheduled page，用于查看、暂停、恢复、编辑和删除任务。
- 任务可按具体时间或“早上、下午、晚上”等时间窗口运行；监控任务可以搜索网页、检查已连接应用，只有在有值得报告的变化时通知用户。
- 功能面向 Plus、Pro、Business、Enterprise 滚动推出。
- Pulse 将被 sunset，主动更新迁移到 scheduled tasks。

来源：
- https://help.openai.com/en/articles/6825453-chatgpt-release-notes

播报角度：
- 这不是单纯提醒事项功能，而是 ChatGPT 从对话入口变成“个人自动化和监控入口”的一步。
- 和 agent、browser、connectors、memory 等方向合在一起看，用户界面层的竞争正在从“谁回答得好”转向“谁能可靠地替用户盯住事情”。

### 5. 阿里云/Qwen：Qwen-Robot Suite 在 6 月 17 日发布，集中押注物理世界智能

- Alibaba Cloud Community 2026-06-17 发布 Qwen-Robot Suite 文章。
- 套件包含三个基础模型：
  - Qwen-RobotNav：导航模型，统一指令跟随、目标搜索、目标跟踪、自动驾驶等任务。
  - Qwen-RobotManip：机器人操作 VLA 模型，解决跨 embodiment 的动作表示和数据规模化。
  - Qwen-RobotWorld：视频世界模型，用自然语言作为动作接口，预测物理世界未来状态。
- Qwen-Robot Suite 文章称，视觉语言模型能理解和规划，但不能直接给出物理控制命令，套件目标是把视觉语言表示空间桥接到物理动作。
- Qwen-RobotNav 基于 Qwen3-VL，训练 1560 万样本，统一五类导航任务；暴露视觉 token 预算、时间衰减、相机权重、帧采样方式等可控观察参数。
- Qwen-RobotManip 基于 Qwen-VL，使用开放机器人数据和人类示范视频，构建约 38100 小时预训练语料；用统一 80 维状态动作表示、相机坐标系末端执行器 delta pose、in-context policy adaptation 做跨机器人对齐。
- Qwen-RobotWorld 使用自然语言统一 20+ robot embodiments 和 500+ action categories，在 860 万 video-text pairs、2 亿+ frames 上训练，强调多视角一致生成和物理合理性。

来源：
- https://www.alibabacloud.com/blog/qwen-robot-suite-a-foundation-model-suite-for-physical-world-intelligence_603262
- https://www.alibabacloud.com/blog/qwen-robotnav-a-scalable-navigation-model-designed-for-an-agentic-navigation-system_603266
- https://www.alibabacloud.com/blog/qwen-robotmanip-alignment-unlocks-scale-for-robotic-manipulation-foundation-models_603267
- https://www.alibabacloud.com/blog/qwen-robotworld-boundless-worlds-for-embodied-agents_603268

播报角度：
- 这是国内模型公司从“文本/多模态能力榜单”走向机器人任务分层的一次集中发布。
- 核心技术问题是“具身数据异构且昂贵”，Qwen 的答案是语言接口、统一动作表示和人类视频到机器人演示的合成。

### 6. Z.AI / GLM：GLM-5.2 发布，主打 100 万 token 长上下文与长程任务

- Hugging Face 2026-06-17 发布 Z.AI 的 GLM-5.2 文章。
- GLM-5.2 是面向 long-horizon tasks 的旗舰模型。
- 主打稳定 100 万 token context、多个思考强度的代码能力、IndexShare 架构、改进 MTP speculative decoding，以及 MIT 开源许可。
- 文章称 IndexShare 在 1M context 时将 per-token FLOPs 降低 2.9 倍；MTP 接受长度提升最多 20%。
- 基准方面，文章称 GLM-5.2 在 FrontierSWE、PostTrainBench、SWE-Marathon 等长程编程任务上是排名最高的开源模型之一。

来源：
- https://huggingface.co/blog/zai-org/glm-52-blog

播报角度：
- 开源模型竞争不再只是“聊天评测”，而是长上下文、长时间工具使用、工程交付和 agentic RL。
- 中国模型在开源许可与国际开发者分发上继续用 Hugging Face 获取开发者注意力。

### 7. Hugging Face：Agentic Resource Discovery，给 agent 找工具、技能、其他 agent 的开放规范

- Hugging Face 2026-06-17 发布 Agentic Resource Discovery: Let agents search for tools, skills, and other agents。
- ARD 是一个开放草案规范，参与者包括 Microsoft、Google、GoDaddy、Hugging Face 等。
- 目标是给 MCP 工具、Skills、A2A agent 等能力提供发现层，使 agent 在运行时搜索能力，而不是依赖预安装。
- 规范包含静态 manifest `ai-catalog.json` 和动态 registry API `POST /search`。
- Hugging Face Discover Tool 是 ARD 的参考实现，提供对 Skills、ML applications、MCP Servers 等的搜索。

来源：
- https://huggingface.co/blog/agentic-resource-discovery-launch

播报角度：
- 这说明 agent 工程进入“能力发现与治理”阶段：不是模型知道所有工具，而是外部 registry 提供发布者、标签、合规声明、代表查询等信号。
- 对企业部署，ARD 解决的是规模化工具目录、权限边界和运行时选择问题。

### 8. 中国资本市场：上交所发布人工智能大模型企业适用科创板第五套上市标准审核指引

- 上海证券交易所 2026-06-17 发布《上海证券交易所发行上市审核规则适用指引第10号——人工智能大模型企业适用科创板第五套上市标准》，文号上证发〔2026〕65号，发布和生效日期均为 2026-06-17。
- 文件目的：落实证监会关于科创板设置科创成长层、增强制度包容性适应性的意见；支持尚未形成一定收入规模的优质人工智能大模型企业在科创板发行上市。
- 上交所新闻稿称，大模型是人工智能产业链的主要技术引擎，企业需要持续高强度技术开发、算力投入及引进专业人才，已处于大规模商业化关键时期。
- 指引把阶段性成果明确为“申报时至少有一个大模型产品已完成上线发布并实现规模化应用”；主营业务范围包括人工智能大模型的自主研发、模型服务或者模型应用，支持通用大模型和行业专用模型。

来源：
- https://www.sse.com.cn/lawandrules/sselawsrules2025/stocks/review/firstepisode/c/c_20260617_10822578.shtml
- https://www.sse.com.cn/aboutus/mediacenter/hotandd/c/c_20260617_10822584.shtml

播报角度：
- 这是国内 AI 大模型企业融资路径的制度化信号。
- 核心判断：监管层承认大模型公司“资本开支前置、商业化滞后”的产业特征，但也要求已有上线产品和规模化应用，避免纯故事融资。

### 9. 中国监管：中央网信办“涉AI应用乱象举报专区”与 AI+通信三年计划

- 中央网信办举报中心 2026-06-12 开设“涉AI应用乱象举报专区”，配合“清朗·整治AI应用乱象”专项行动。
- 受理范围包括未履行大模型备案登记、平台安全和审核过滤不足、训练语料安全、数据投毒、合成内容标识落实不到位、滥用 AI 技术、开源模型安全管理不到位，以及 AI 魔改经典、虚假不实信息、假冒仿冒、侵害未成年人权益等。
- 中国政府网英文版 2026-06-10 报道，工信部发布 2026-2028 三年计划，推动 AI 与信息通信行业融合；目标到 2028 年信息通信网络初步达到高水平自治智能，提出 30+ 高价值用例、典型应用和专业智能体，以及都市区 1 毫秒算力接入覆盖率至少 75%。

来源：
- https://www.cac.gov.cn/2026-06/12/c_1782660002371256.htm
- https://english.www.gov.cn/news/202606/10/content_WS6a296017c6d00ca5f9a0b876.html

播报角度：
- 中国 AI 政策正在一手鼓励融资和基础设施，一手强化应用乱象治理。
- 对企业来说，产品备案、内容标识、数据安全、开源模型管理都将成为商业化必备项。
