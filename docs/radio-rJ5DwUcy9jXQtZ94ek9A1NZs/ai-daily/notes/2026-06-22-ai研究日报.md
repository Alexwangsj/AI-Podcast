## 重点事实与来源

### 1. Z.ai GLM-5.2：1M token 长上下文和长程 coding agent

来源：

- Z.ai 官方博客：GLM-5.2: Built for Long-Horizon Tasks  
  https://z.ai/blog/glm-5.2
- GLM-5.2 Hugging Face weights  
  https://huggingface.co/zai-org/GLM-5.2
- GLM-5 GitHub  
  https://github.com/zai-org/GLM-5
- Business Insider 对 GLM-5.2 的报道  
  https://www.businessinsider.com/what-is-glm-5-2-chinese-ai-coding-model-2026-6

关键事实：

- Z.ai 在 2026-06-16 发布 GLM-5.2，定位是面向 long-horizon tasks 的旗舰模型。
- 官方材料强调 GLM-5.2 提供 “solid 1M-token context”，目标是稳定支撑长程工作，而不是只扩大 prompt 长度。
- 长上下文训练覆盖 coding-agent 场景：大规模实现、自动化研究、性能优化、复杂调试。
- 技术上引入 IndexShare，在 1M 上下文长度下减少 indexer 相关计算；官方材料称每 token FLOPs 降低约 2.9 倍。
- 改进 MTP layer 做 speculative decoding，官方材料提到 acceptance length 最高提升 20%。
- 长上下文推理侧强调 KV-cache、长上下文 kernel、CPU cache 管理、请求调度、runtime execution paths。
- Agentic RL 训练侧使用 slime 作为从训练到大规模 inference rollout 的基础设施层，支持 white-box rollout、black-box rollout、compact trajectory、sub-agent workflow。
- 官方称在 FrontierSWE、PostTrainBench、SWE-Marathon 等长程工程基准上，GLM-5.2 是最高排名的开源模型；在 Terminal-Bench 2.1、SWE-bench Pro 等标准 coding benchmark 上也显著超过 GLM-5.1。
- 权重公开在 Hugging Face 和 ModelScope；本地部署支持 transformers、vLLM、SGLang、xLLM、ktransformers 等框架。

技术解读：

- GLM-5.2 的核心信号是“长上下文工程化”：上下文窗口、注意力计算、KV-cache、speculative decoding、RL rollout 和 agent task training 被一起处理。
- 这和很多只把上下文长度当参数的发布不同。长程 agent 真正卡住的地方是：能否在几十万 token 的代码库、日志、测试结果和中间计划里保持任务目标，不被噪声带偏。
- 对开发者的意义：如果官方 benchmark 能被真实使用验证，开源/可本地部署模型会更适合私有代码库、离线安全环境和高成本 coding agent workload。

### 2. Qwen-Robot Suite：Qwen 从数字世界走向物理 AI

来源：

- Alibaba Cloud Community：Entering the Physical AI Era: Introducing the Qwen-Robot Suite  
  https://www.alibabacloud.com/blog/entering-the-physical-ai-era-introducing-the-qwen-robot-suite_603261
- Qwen-RobotNav  
  https://www.alibabacloud.com/blog/qwen-robotnav-a-scalable-navigation-model-designed-for-an-agentic-navigation-system_603266
- Qwen-RobotManip  
  https://www.alibabacloud.com/blog/qwen-robotmanip-alignment-unlocks-scale-for-robotic-manipulation-foundation-models_603267
- Qwen-RobotWorld  
  https://www.alibabacloud.com/blog/qwen-robotworld-boundless-worlds-for-embodied-agents_603268

关键事实：

- Alibaba Cloud 在 2026-06-17 介绍 Qwen-Robot Suite，这是阿里首个基于 Qwen 的机器人基础模型套件。
- 套件包含三个核心模型：
  - Qwen-RobotManip：Vision-Language-Action 模型，面向机器人操作。
  - Qwen-RobotNav：Vision-Language-Navigation 模型，面向导航和 embodied question answering。
  - Qwen-RobotWorld：视频世界模型，面向具身智能和物理世界未来状态预测。
- 官方材料称套件目标是把大多模态模型从文本、图像、语音理解扩展到物理动作和环境交互。
- Qwen-RobotManip 基于 Qwen3.5-4B VL，使用超过 38,000 小时开源数据，来自机器人仓库、人类操作视频和人类到机器人合成数据。
- Qwen-RobotNav 基于 Qwen3-VL，训练语料包含 1,560 万样本，覆盖轨迹规划和视觉语言推理。
- 套件已经进入机器人行业部分阿里云企业客户试点。

技术解读：

- Qwen-Robot Suite 的价值在于把“语言指令、视觉理解、动作表示、环境动态”拆成导航、操作、世界模型三个层次。
- 机器人模型难点不在是否理解文字，而在不同 embodiment、相机视角、动作空间、物理约束和数据来源之间如何对齐。
- 如果 Qwen 的路线可行，国内大模型公司会从聊天和 coding agent 进一步进入工业机器人、仓储、服务机器人、具身问答等场景。

### 3. Hugging Face Agentic Resource Discovery：agent 开始搜索工具、skills 和 MCP server

来源：

- Hugging Face：Agentic Resource Discovery: Let agents search for tools, skills, and other agents  
  https://huggingface.co/blog/agentic-resource-discovery-launch
- ARD GitHub specification  
  https://github.com/huggingface/agentic-resource-discovery

关键事实：

- Hugging Face 在 2026-06-17 发布 Agentic Resource Discovery（ARD）。
- ARD 规范定义两件事：
  - 静态 manifest：`ai-catalog.json`，发布者可以放在 well-known URL。
  - 动态 registry API：`POST /search`，提供实时、排序后的资源发现。
- Hugging Face Discover Tool 是 ARD 参考实现，可以搜索 Skills、ML apps、MCP Servers、Spaces。
- Hub 的 semantic search 支持 `agents=true`，返回按 agent-oriented metadata 排序的 Spaces。
- Discover adapter 会筛选 runtime stage 为 `RUNNING` 的 Spaces，并按请求返回不同 media type：
  - `application/ai-skill`
  - `application/mcp-server+json`
  - `application/vnd.huggingface.space+json`
- 对 Space 中的 `agents.md`，Discover 会包装成 skill consumer 可安装或加载的技能格式。
- CLI 用法示例包括 `hf discover search "Fine tune a language model"` 和按 MCP 类型搜索资源。

技术解读：

- 智能体如果要从 demo 走向工作流，不能只靠开发者手动装插件。它需要在任务中发现能力：找工具、找 MCP 服务、找另一个 agent、判断资源是否在线。
- ARD 的价值类似“给 agent 的 package registry + search API”。如果它被更多平台采纳，agent 编排会从固定工具列表变成动态资源发现。
- 风险是安全边界会更复杂：搜索到的工具能不能信、权限如何授予、执行环境如何隔离、manifest 如何防投毒，都将变成 agent 平台的基础问题。

### 4. GitHub Copilot usage metrics API：AI credits 进入开发者工具治理层

来源：

- GitHub Changelog：AI credits consumed per user now in the Copilot usage metrics API  
  https://github.blog/changelog/2026-06-19-ai-credits-consumed-per-user-now-in-the-copilot-usage-metrics-api/

关键事实：

- GitHub 在 2026-06-19 更新 Copilot usage metrics API。
- 新增 `ai_credits_used` 字段，用于用户级报告，表示该用户在 Copilot 活动中消耗的 AI credits 总量。
- 字段适用于 enterprise 和 organization 层面的 single-day `users-1-day` 与 28-day `users-28-day` 报告。
- GitHub 明确说明该字段是 consumption signal，不是账单总额；当前也不按 feature、model 或 surface 拆分。

技术解读：

- 这看似是计费指标，实际是 AI coding 工具进入“成本可观测性”的标志。
- 对团队管理者来说，下一步不是只问 Copilot 有没有提升效率，而是把 credits、PR、code review、CI、bug rate、开发周期放在一起看。
- 对 agent 产品来说，成本会越来越成为设计约束：是否长上下文、是否多模型路由、是否重试、是否跑长程任务，都需要被指标化。

### 5. Google Gemini API deprecations：图像 preview 模型迁移窗口临近

来源：

- Google AI for Developers：Gemini deprecations  
  https://ai.google.dev/gemini-api/docs/deprecations

关键事实：

- Gemini deprecations 页面列出 GA 和 preview 模型的关闭计划。
- Gemini 3 模型中：
  - `gemini-3.1-flash-image-preview` 计划 2026-06-25 关闭，推荐替代为 `gemini-3.1-flash-image`。
  - `gemini-3-pro-image-preview` 计划 2026-06-25 关闭，推荐替代为 `gemini-3-pro-image`。
  - `gemini-3.5-flash` 发布于 2026-05-19，当前没有 shutdown date。
  - `gemini-3.1-flash-image` 和 `gemini-3-pro-image` 发布于 2026-05-28，当前没有 shutdown date。
- Google 说明 shutdown 后 endpoint 不再可用，具体关闭日期会提前通知。

技术解读：

- 这不是大模型能力新闻，但对开发者很实在：如果应用还依赖 Gemini 图像 preview endpoint，本周需要迁移。
- 这也说明多模态模型进入产品稳定期：preview 到 GA 替换，不只是名字变化，通常还会影响质量、速率、配额、价格、错误行为和 safety filter。

技术解读：

- 模型退役提醒则说明开发者要区分 ChatGPT 产品模型和 API 模型；企业自动化不要把 ChatGPT UI 的模型名当稳定 API 依赖。

### 7. HPE / NVIDIA AI Factory：企业 agent 从实验走向可控生产

来源：

- HPE Discover Las Vegas 2026  
  https://www.hpe.com/us/en/discover/lasvegas.html
- ITPro：HPE AI Factory adds features to improve your experience with agents  
  https://www.itpro.com/technology/artificial-intelligence/hpe-ai-factory-adds-features-to-improve-your-experience-with-agents

关键事实：

- HPE Discover Las Vegas 2026 于 2026-06-15 至 06-18 举行，AI、cloud、networking 是核心议题。
- ITPro 报道称，HPE 和 NVIDIA 在 HPE Discover 2026 更新 HPE AI Factory。
- HPE AI Factory 将提供 NVIDIA Agent Toolkit，包括 NVIDIA Nemotron open models、NVIDIA NemoClaw、NVIDIA OpenShell secure runtime。
- HPE Private Cloud AI 增加 HPE ProLiant Compute DL394 Gen12 servers，配 NVIDIA Vera CPU。
- HPE/Zerto 增加能力，用于在 rogue agent 执行不应执行动作时通过 continuous data protection 回滚到 clean state。
- HPE Private Cloud AI 支持 secure local agent registration，使企业能在集中治理下批准 AI models、skills 和 tools。
- HPE Alletra Storage MP X1000 用于数据准备、元数据和治理策略；报道提到可提升 token throughput 最高 20%。

技术解读：

- 企业 agent 的难点不只是模型效果，而是 runtime、权限、审计、数据准备、回滚、工具注册和可观测性。
- HPE/NVIDIA 的方向说明 agent stack 正在基础设施化：模型只是一个层，下面需要数据、网络、GPU、storage、policy、rollback 和治理系统。
- 对企业私有部署来说，这比单纯“接一个 API”更贴近真实生产。
