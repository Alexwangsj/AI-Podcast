# AI研究日报 2026-06-30 研究笔记

时间窗口：主要关注 2026-06-29 至 2026-06-30，可作为技术背景少量引用过去一周内仍在发酵、且影响今天模型可用性或工程路线的事件。

## 1. 技术主线概览

1. 前沿模型竞争继续从“单次问答能力”转向“长程任务执行”。OpenAI 的 GPT-5.6 系列预览强调 `max` 推理努力、`ultra` 多子智能体模式、Terminal-Bench 2.1、GeneBench、ExploitBench/ExploitGym 等长程编码、生物和安全任务；中国侧 GLM-5.2、DeepSeek V4、MiniMax-M3 等继续把 1M 上下文、Coding Agent、OpenAI/Anthropic 兼容 API 作为核心工程卖点。
2. 推理基础设施正在成为模型路线的一部分。OpenAI 与 Broadcom 的 Jalapeno 推理芯片把 LLM kernel、内存移动、网络和 serving pattern 纳入芯片设计；Google Gemini API 正在关停旧 Veo 模型并要求迁移到 3.1；DeepSeek 将 `deepseek-chat` 与 `deepseek-reasoner` legacy 名称路由到 V4 Flash 并计划 2026-07-24 退役。
3. 研究方向上，“世界模型”和“模型协作”变得更具体。arXiv 6 月 29 日新稿里，多个工作讨论 LLM agent 的未来状态预测、参数化 world model 对 hallucinated state 的约束、多智能体稀疏通信、VLA 模型冗余、SpikeVLA 和长程操控。

## 2. 模型与能力升级

### OpenAI：GPT-5.6 系列有限预览

- OpenAI 开始有限预览 GPT-5.6 系列：Sol 为旗舰模型，Terra 定位日常工作平衡模型，Luna 定位低成本快速模型。Terra 被描述为达到接近 GPT-5.5 的竞争性能且成本约低一半，Luna 是最低成本强能力版本。
- 技术点：Sol 引入 `max` reasoning effort；`ultra` 模式使用子智能体加速复杂任务；重点指标覆盖 Terminal-Bench 2.1、GeneBench v1、ExploitBench 和 ExploitGym。
- 可用性：OpenAI 表示因与美国政府沟通，先面向少量 trusted partners 预览，再计划未来数周更广泛开放。这是一个政策影响模型访问的技术可用性事件，但本期只压缩呈现。
- 来源：https://openai.com/index/previewing-gpt-5-6-sol/

### xAI：Grok 4.5 私测与月度模型节奏

- Elon Musk 在 X 上称 Grok 4.5 已进入 SpaceX 与 Tesla 私有 beta，基于约 1.5T V9 foundation model，并在 supplemental training 中加入 Cursor 数据。多家媒体跟进称 xAI 计划今年余下时间每月推出从头训练的新模型。
- 技术意义：如果属实，xAI 重点转向编码/工程数据、内部高压工程场景测试和更密集模型迭代。但当前缺少公开基准，需避免把内部声明当作已验证能力。
- 来源：https://x.com/elonmusk/status/2071184354756477041
- 参考报道：https://www.ndtvprofit.com/business/musk-unveils-grok-4-5-private-beta-claims-ai-rivals-claude-opus-11698558

### 中国模型生态：GLM-5.2、DeepSeek V4、MiniMax-M3

- Z.ai/智谱 GLM-5.2 的官方技术说明强调 1M 上下文、长程 Coding Agent、长任务交付和开源权重；文档称最大输出 128K tokens，并面向复杂工程、论文复现、代码生成视频闭环等场景。

## 3. 工程与基础设施

### OpenAI + Broadcom：Jalapeno 推理芯片

- OpenAI 与 Broadcom 发布 Jalapeno，称其是 OpenAI 第一代 Intelligence Processor，面向 LLM inference 从零设计，而非把通用加速器改造成推理芯片。
- 设计重点：基于 ChatGPT、Codex、API 和未来 agentic products 的真实 serving pattern，围绕 kernels、memory movement、networking 和 serving patterns 优化。
- 路线：OpenAI 称早期测试显示每瓦性能显著优于当前 SOTA；从设计到生产 9 个月；将与数据中心伙伴在多代路线中以 gigawatt scale 部署。
- 来源：https://openai.com/index/openai-broadcom-jalapeno-inference-chip/

### Google Gemini API：Veo 旧模型退役

- Gemini API changelog 显示，`veo-2.0-generate-001`、`veo-3.0-generate-001`、`veo-3.0-fast-generate-001` 在 2026-06-30 shutdown；开发者需要迁移到 Veo 3.1 preview 或 Gemini Enterprise Agent Platform 的 3.1 GA 模型。
- 技术意义：视频生成 API 已进入更快生命周期，产品集成必须把模型 lifecycle 和迁移机制当成工程基础设施，而不是临时变更。
- 来源：https://ai.google.dev/gemini-api/docs/changelog

### GitHub Copilot：MAI-Code-1-Flash 企业可用

- GitHub 6 月 changelog 显示 Microsoft AI 的内部编码模型 MAI-Code-1-Flash 已面向 Copilot Business 和 Enterprise GA，定位为 purpose-built for coding and optimized for Copilot surfaces。
- 这不是 6 月 29/30 的新发布，但仍是过去几天开发者生态的重要信号：编码模型正在从通用 frontier API，进入 IDE/代码审查/企业权限系统内的专用小模型和模型路由。
- 来源：https://github.blog/changelog/month/06-2026/

## 4. 开源与开发者生态

- Hugging Face 6 月 29 日刊出 Ai2/Allen Institute 的 DiScoFormer 文章。DiScoFormer 用一个 transformer 对有限样本背后的分布同时估计 density 和 score，在单次 forward pass 中完成，不需要为每个新分布重新训练；模型用 shared backbone + density/score 两个输出头，并利用 score 与 log-density 梯度关系形成 label-free consistency loss。
- 技术意义：这类工作不直接面向聊天模型，但对扩散模型、生成建模和不确定性估计有底层启发，尤其是“一个模型跨分布估计”的方向。
- 来源：https://huggingface.co/blog/allenai/discoformer

## 6. 研究论文与机器人/具身智能

### Brain2Qwerty：非侵入式脑信号到文本

- Nature Neuroscience 2026-06-29 发表 “Noninvasive decoding of typed sentences from human brain activity”。研究提出 Brain2Qwerty，用 EEG 或 MEG 记录人在键盘上输入短句时的脑活动，并从脑信号解码文本生产。
- 数据与结果：35 名健康志愿者；MEG 平均字符错误率 29%，EEG 平均 65%；最佳 MEG 参与者字符错误率 18%。模型包含卷积模块、句子级 transformer 和预训练语言模型纠错。
- 技术意义：非侵入式 BCI 距离临床可靠通信仍有距离，但 AI 架构开始把脑信号局部时窗、序列模型和语言模型结合起来，逐步缩小侵入式和非侵入式方法差距。
- 来源：https://www.nature.com/articles/s41593-026-02303-2
- Meta 介绍页：https://ai.meta.com/blog/brain2qwerty-brain-ai-human-communication/

### LLM Agent 世界模型与规划

- arXiv 6 月 29 日 cs.AI new listings 包含多个围绕 agent world model 的工作：
  - Internalizing the Future：用 World Model Agentic Mid-Training、Format-Eliciting SFT、Foresight-Conditioned RL 让单个自回归模型生成未来状态 rollout 和成功估计。
  - Grounded Iterative Language Planning：用小型参数化 world model 给 API LLM 的 imagined next state 做一致性校验，减少 hallucinated state 在长程规划中的传播。
  - Graph World Models：分析图结构 world model 的 rollout error 和 long-horizon divergence。
- 来源：https://arxiv.org/list/cs.AI/new
- GILP 论文：https://arxiv.org/html/2606.27806

### 机器人/VLA

- arXiv cs.RO 6 月 29 日新稿中，VLA 方向集中讨论“能否更小、更省、更稳”：
  - Drop-Then-Recovery 研究 VLA transformer block 冗余。
  - SpikeVLA 使用 spiking neural networks 降低 VLA 推理延迟和能耗。
  - S2-VLA 用 state-space guided 机制应对长程操控中的误差积累。
  - 云原生仿真基础设施论文把训练、评估、轨迹采集、数据管理和云服务统一到可复现平台。
- 来源：https://arxiv.org/list/cs.RO/new
- SpikeVLA：https://arxiv.org/html/2606.27807
- S2-VLA：https://arxiv.org/html/2606.27872

## 7. 政策/商业补充

- 模型访问限制继续影响开发者路线。OpenAI GPT-5.6 预览先给 trusted partners；中国开源模型在此背景下获得更多全球开发者关注，尤其是 GLM-5.2 在长上下文、代码和开放权重上的定位。

## 结论

今天最值得关注的不是某一个模型名字，而是三个工程方向同时加速：长程 agent 需要更长上下文和更强状态校验；推理成本开始被芯片、稀疏注意力、模型生命周期和兼容 API 共同决定；具身智能与 BCI 研究正在把 transformer、world model、VLA 和信号处理推向更具体的系统形态。
